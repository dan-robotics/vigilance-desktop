🚀 Vigilance v1.0.1 — The Intelligence Update

Vigilance has officially moved from prototype to a production-grade security engine. This release introduces Real-time Protocol Decoding, GeoIP Intelligence, and AI-Powered Threat Analysis, transforming raw network data into actionable security intelligence.

💎 Key Highlights

  • Kernel Transparency: Replaced the generic "Guardian Kernel" label with specific decoding for ICMP, IGMP, GRE, ESP, AH, OSPF, PIM, VRRP, SCTP — and a PROTO-N fallback for any unknown IP protocol number.
  • GeoIP Intelligence: Live async location lookups identify the City, Country, and ISP/Organization (e.g. AWS, Korea Telecom, Rostelecom) for every connection and detection card.
  • AI Guardian: Every new detection automatically triggers a Gemini 2.0 Flash explanation. When AI is unavailable, a built-in local engine generates context from GeoIP, port semantics, and heuristic data — no API key required.
  • Detection Deduplication: 60-second cooldown per IP:reason pair eliminates duplicate alerts from bidirectional packet capture.
  • Static-Width Table: Process Activity Feed now uses a fixed-column layout with protocol badges that stack vertically — no more horizontal scrolling.
  • Universal Compatibility: Optimized builds for all modern macOS (Universal, Apple Silicon, Intel) and Windows architectures.

🔒 Security & Permissions (macOS)

To capture live network traffic, Vigilance requires BPF (Berkeley Packet Filter) access.
Recommended: Grant BPF access to your user once per boot — avoids running the entire app as root: 

sudo chown $(whoami) /dev/bpf*

Then launch normally: open Vigilance.app (production) or npm run tauri dev (dev).

Alternative — run the binary directly as root:

sudo /Applications/Vigilance.app/Contents/MacOS/vigilance
